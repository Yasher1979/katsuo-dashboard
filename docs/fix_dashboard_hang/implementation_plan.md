# ダッシュボード表示停止に関する根本的修正計画

最近ダッシュボードの表示が止まってしまう問題について、コードを詳細に調査した結果、いくつかの「根本的な不安定要因」が見つかりました。これらを修正し、確実に「動く」状態に戻します。

## ユーザー確認が必要な事項

> [!IMPORTANT]
> **外部ライブラリの影響**: 
> 現在、Chart.js などのライブラリを「常に最新版」で読み込む設定になっています。これが原因で、ライブラリ側のアップデート（v4への移行など）により、既存のコードと衝突して停止している可能性が非常に高いです。これを **「安定版（v3.9.1）」に固定** します。

## 根本原因の分析と提案

### 1. ライブラリのバージョン不一致（最有力）
- **現状**: `chart.js`（現在v4が主流）と `chartjs-adapter-moment@1.0.1`（v3用）が混在している可能性があります。
- **解決策**: ライブラリのバージョンを明示的に固定し、互換性を担保します。

### 2. エラー情報の「沈黙」
- **現状**: 初期化の早い段階でエラーが出ると、エラーメッセージが表示される前に画面がロック（スプラッシュ画面のまま）されます。
- **解決策**: グローバルエラーハンドラー（`window.onerror`）を追加し、何が起きてもエラー内容を画面に表示するようにします。

### 3. ストレージ利用の安全性
- **現状**: iPhoneのプライベートブラウザなど、`localStorage` が制限されている環境でコードがクラッシュする可能性があります。
- **解決策**: `try-catch` を強化し、ストレージに触れなくても動作するようにガードを入れます。
