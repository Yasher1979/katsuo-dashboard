# 低トークン運用ルール

このファイルは、相場更新作業を速く・安く・短く進めるためのルールです。
作業品質を落とさず、無駄な探索と長文説明を減らします。

## 基本姿勢

- まず `AGENTS.md` と関連ルール文書を読む。
- 既に制度化されたルールを再説明しすぎない。
- 大量ファイルを広く読む前に、対象日付・港・船名・サイズで絞る。
- 作業ログは必要最小限。最終報告は「変更点・検証結果・残作業」だけを優先する。
- 不明点が1つだけなら、短く1問だけ聞く。

## 原始人プロンプト

ユーザーは短い指示でよい。細かい文章にしなくてよい。

例:

```text
焼津 7/14 追加。海旋だけ。検証まで。
```

```text
枕崎 7/15 更新。画像通り。デプロイ前まで。
```

```text
山川 グラフ変。最新日だけ見て。
```

```text
原始人: 焼津おかしい。2.5だけ。一本釣り除外。
```

## AI側の省トークン手順

1. 最初に対象を絞る。
   - 港
   - 日付
   - 船名
   - サイズ
   - 相場CSVか入札予定か
2. `rg` で対象だけ検索する。
3. 必要なファイルだけ読む。
4. CSV更新後はJSON再生成と検証を優先する。
5. 長い説明より、検証結果を短く出す。

## 探索を広げる条件

次の場合だけ広く検索する。

- 対象船名が見つからない。
- CSVとJSONの内容が食い違う。
- 表示画面がローカルデータと違う。
- デプロイ済み画面だけ異常が残る。
- 既存ルールにない市場・漁法・サイズが出てきた。

## 最終報告テンプレート

```text
更新しました。

変更:
- ...

検証:
- ...

残り:
- ...
```

残りがなければ `残り` は省略する。

## 禁止

- 毎回リポジトリ全体を長く読み直さない。
- 既に分かっているルールを長文で再説明しない。
- 不要な背景説明を増やさない。
- ユーザーが急いでいるときに、確認質問を連発しない。
- 推測で相場データを作らない。

## 省トークン合図

ユーザーが次の言葉を使ったら、特に短く動く。

- `原始人`
- `短く`
- `最短`
- `いつもの手順`
- `検証まで`
- `デプロイ前まで`

この場合、AIは説明を減らし、作業と検証を優先する。
