# 枕崎データ不整合および表示バグの修正計画

枕崎の最新データが正しく「2026-02-10」として扱われ、全てのサイズが一覧表示されるように修正します。

## 概要
以前の機能デモ用に追加した 2/17 のダミーデータが、枕崎では「1.8kg上」のみに存在するため、ダッシュボードの「最新」判定が 2/17 になり、結果として他サイズが表示されない状態になっています。これを実データに合わせて整理します。

## 変更内容

### 1. データ層
- **[MODIFY] data/market_input.csv**
  - 焼津のデモ用 2/17 データを削除します。
- **[RUN] scripts/katsuo_fetcher.py**
  - CSVから最新の JSON を再生成します。
- **[MODIFY] data/katsuo_market_data.json** (スクリプト実行により更新)
  - 手動追加した枕崎の 2/17 データを排除します。

### 2. 表示層 (念のための調整)
- **[MODIFY] web/dashboard.js**
  - `getLatestPortData` 内の最新日付取得ロジックを見直し、特定サイズしかデータがない極端なケースでも適切なエラーハンドリングまたは表示が行えるか確認します。

## 検証方法
1. ブラウザを更新し、枕崎のカードの日付が「2026-02-10」であること。
2. 全てのサイズ区分が表示されていること。
