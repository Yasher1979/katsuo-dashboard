# AIアシスタント不在時の運用・引き継ぎマニュアル

## 1. はじめに
本ドキュメントは、AIアシスタント（Antigravity等）が何らかの理由で利用できなくなった場合に、本システム（鰹相場ダッシュボード）を他のAI、人間のエンジニア、あるいはユーザーご自身で継続的に運用・更新していくための手順と仕様をまとめたものです。

本システムは、特定のサービスやプロプライエタリ（非公開）な技術に依存しない、**完全なオープン・スタンダード（HTML, CSS, Vanilla JavaScript）**で構築されています。そのため、ベンダーロックイン（特定のAIがいないと動かない状態）は発生せず、ファイルを移行するだけで即座に他者へ引き継ぎが可能です。

## 2. システムの基本構造
特殊なサーバーやデータベースは一切使用しておらず、GitHub Pages等の静的ホスティングで稼働します。

*   **フロントエンド**: `web/index.html`, `web/index.css`, `web/dashboard.js`
*   **データソース (マスター)**: 
    *   `data/market_input.csv` (相場価格の元データ)
    *   `data/market_insights.json` (相場日誌・ニュースデータ)
    *   `data/bid_schedule.json` (入札予定データ)
*   **データ処理スクリプト**: `scripts/` フォルダ内のPythonスクリプト群（CSVから表示用JSONを生成）

## 3. 万が一の際の引き継ぎ方法

### パターンA：別のAI（ChatGPT, Claude等）に運用を任せる場合（推奨）
最も簡単で素早い方法です。
1. PC上にあるこのシステム全体のフォルダ（`Antigravity（鰹相場グラフ）`）をZipファイルなどに圧縮します。
2. 新しいAIアシスタントにそのファイルをアップロードし、以下のように指示します。
   > 「このフォルダのデータを使って鰹の相場ダッシュボードを運用しています。本日の相場データ（〇〇円）をCSVに追加し、表示を更新してください」
3. AIがコードの構造を理解し、適切なファイルの書き換えを代行してくれます。

### パターンB：人間のエンジニアに引き継ぐ場合
フォルダ一式と、データを保管している「GitHub」のアカウント権限をエンジニアに譲渡してください。
フロントエンドの標準的な知識を持つエンジニアであれば、数十分で構造を理解し、更新や改修作業を開始できます。

### パターンC：ユーザー様ご自身で手動更新する場合
AIやエンジニアを頼らずとも、手動でのデータ更新は可能です。

1. **相場データの更新**:
   *   `data/market_input.csv` をExcel等で開きます。
   *   一番下の行に、新しい日付と各港の価格・水揚量データを追記して上書き保存します。
2. **相場日誌の更新**:
   *   `data/market_insights.json` をメモ帳等で開きます。
   *   既存のフォーマットに倣って、新しい日付とニュースを追記します。
3. **表示データの生成**:
   *   ターミナル（コマンドプロンプト）で `python scripts/process_market_data.py` を実行すると、CSVから表示用のデータが自動生成されます。
4. **全世界への公開（デプロイ）**:
   *   GitHub Desktop等のアプリを使用し、「Commit」して「Push」するだけで、インターネット上の画面が最新状態に更新されます。

## 4. 最後に
このダッシュボードのデータは、長年の相場の歴史を刻んだ会社にとっての「重要な資産」です。
どのような状況になっても、このフォルダさえ大切にバックアップ・保管していただければ、システムとデータが失われることはありません。安心して運用を継続してください。
